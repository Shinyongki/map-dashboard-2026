경상남도 광역지원기관  |  최종 업데이트: 2025


# 1. 시스템 정의
경남 59개 수행기관, 270개 복지자원, 18개 시군의 기상·재난 데이터, 공문서를 하나의 플랫폼에서 통합 관리하고, LLM 기반 지능형 분석으로 광역지원기관의 의사결정과 현장 사회복지사의 실무를 동시에 지원하는 통합 인텔리전스 시스템.

| --- | --- |
| 시스템 명칭 | 경남 노인맞춤돌봄 통합 인텔리전스 시스템 |
| 운영 주체 | 경상남도 광역지원기관 (경남사회서비스원) |
| 대상 기관 수 | 59개 수행기관 / 18개 시군 |
| 사용자 | 광역지원기관 담당자 / 현장 사회복지사 (기관코드 인증) |



# 2. 핵심 목표
59개 기관 월별 현황 데이터의 정밀 집계 및 이상 신호 자동 감지
읍면동 단위 복지자원 연결망 구축 및 자원 경로 추천
기상·재난 데이터와 돌봄 현황의 실시간 결합 분석
공문서 기반 LLM 질의응답으로 현장 유선 문의 대폭 감소
광역 담당자의 판단·보고 업무 자동화 (AI 초안 생성)
시스템 사용이 누적될수록 고도화되는 지식 베이스 구축

# 4. 시스템 3-레이어 구조

## Layer 1. 현황 파악 — 신호 감지
단순 집계를 넘어 시계열 추세 기반 위험 신호를 자동으로 감지하는 레이어.
월별 59개 기관 현황 제출 (종사자/대상자/단기집중)
전월 대비 이상치 자동 플래그 (종사자 20% 이상 변동 등)
기상청 API 폭염·한파 특보와 돌봄 현황 데이터 실시간 결합
재난 발생 이력 누적 → 다음 대응 판단 참조 데이터로 활용
3개월 이상 연속 추세 감지 (종사자 감소, 신규 유입 정체 등)


## Layer 2. 연결 — 자원 경로
270개 복지자원을 읍면동 단위로 구조화하고, 수행기관과의 연계 관계를 지도로 만드는 레이어.
읍면동 단위 복지자원 DB 구축 (노인의료복지시설, 재가서비스 등)
수행기관 ↔ 복지자원 간 연계 관계 매핑
자원 경로 추천: 대상자 상황에 따른 최적 연계 경로 제시
타 시군 우수 자원 참조 → 유선 문의 → 신규 자원 발굴 → DB 업데이트 순환
자원 공백 지역 시각화 (읍면동 단위)


## Layer 3. 지식 — 판단 지원
공문서, 사업안내서, 현황 데이터, 연계 이력이 누적되어 LLM이 참조하는 지식 베이스를 구성하는 레이어.
공문서 RAG 질의응답 (현장 공개 / 광역 관리)
FAQ 캐싱으로 LLM 호출 최소화 (비용 최적화)
광역 전용: 모니터링 브리핑 자동 생성, AI 보고서 초안
현장 공개: 공문 질의응답, 복지자원 경로 안내, 본인 기관 현황 질의
질의응답 이력 누적 → FAQ 고도화 → 공문 작성 방식 개선 피드백


# 5. 데이터 구조

## 5.1 지식 베이스 (RAG DB) 구성

| 월별 기관 현황 | 59개 기관 × 월별 시계열 누적 (종사자/대상자/단기집중) |
| --- | --- |
| 복지자원 정보 | 270개소 기본정보 + 연계 이력 + 서비스 유형 태그 |
| 기상/재난 이력 | 기상청 특보 이력 + 재난 발생 이력 + 대응 결과 |
| 공문서 아카이브 | 발송 공문 전체 + 요약 + FAQ + 유사 공문 연결 |
| 사업안내서 | 노인맞춤돌봄서비스 사업안내서 및 관련 지침 |
| 모니터링 보고서 | 현장 방문 결과 누적 (지적사항, 개선 이력) |
| 질의응답 이력 | 현장 질의 + 광역 승인 FAQ (고도화 재료) |



## 5.2 기존 데이터 구조 유지
기관 마스터: 시군, 기관명, 기관코드, 거점여부
월별 제출 데이터: 성별/유형별 종사자 및 대상자 수, 단기집중 단계별 현황
배정 인력 데이터: 정부 배정 인원, 변경 이력
지도 데이터: 경남 18개 시군 공간 정보 (SVG Path)


# 6. 핵심 기능 요구사항

## 6.1 공문서 질의응답 시스템
가장 즉각적인 실무 효과를 가져오는 핵심 기능.
광역 담당자가 공문 업로드
LLM이 공문 분석 → 핵심 요약 + 영향 기관 분류 + 예상 FAQ 초안 생성
광역 담당자가 FAQ 검토/승인 (민감한 지침 해석 필터링)
현장 사회복지사에게 공개 활성화
현장 질의 → 유사 FAQ 캐시 매칭 (LLM 미호출) 또는 RAG 검색 후 답변
누적 질의가 FAQ를 고도화하는 순환 구조

LLM 내부 참조: 공문 + 사업안내서 + 관련 지침을 동시 참조하되, 답변은 공문 기준으로 표시


## 6.2 복지자원 연결망
읍면동 단위 자원 지도 (기관 위치, 서비스 유형, 연계 가능 여부)
질의 기반 경로 추천: '이 어르신 상황에 맞는 연계 자원과 경로는?'
타 시군 우수 사례 참조 기능
신규 자원 발굴 시 광역 승인 후 DB 등록 워크플로우


## 6.3 광역 담당자 브리핑
일일 자동 브리핑: 미제출 기관, 이상치 감지, 활성 기상특보, 미승인 FAQ
모니터링 방문 전 기관 브리핑 자동 생성 (3개월 추세 + 이전 지적사항)
AI 보고서 초안: 현황 데이터 기반 월간 보고서, 평가 의견서


## 6.4 기상/재난 + 돌봄 통합 뷰
폭염/한파 특보 시 해당 시군 독거노인 수 + 담당 종사자 현황 자동 결합
'지금 창원시 폭염 상황에서 어느 기관에 먼저 연락해야 하나?' 질의 가능
재난 이력과 서비스 중단 이력 연결 → 다음 재난 대비 가이드


# 7. LLM 비용 최적화 전략
FAQ 캐싱: 동일/유사 질문은 LLM 미호출, 캐시 반환
공문 유효 기간 내 질의응답만 활성화 (기간 만료 후 자동 아카이브)
RAG 청크 최적화: 관련 문서만 정밀 검색하여 컨텍스트 최소화
현장 사용자 질의는 공문/자원 영역으로 범위 제한
